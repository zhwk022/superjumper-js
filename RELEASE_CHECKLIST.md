# 发布检查清单（GitHub Pages）

## 1. 代码与资源

- [ ] `index.html` 引用入口正确（当前应为 `./src/app.js`）
- [ ] 所有资源路径使用相对路径并可在子路径访问
- [ ] 本地运行 `python3 -m http.server 8080` 能完整游玩
- [ ] 菜单、帮助、高分、暂停、过关剧情可正常切换
- [ ] 声音开关与高分榜本地存档正常

## 2. 仓库与分支

- [ ] 已提交并推送到 `main` 分支
- [ ] 仓库包含 `README.md`、`LICENSE`、`.gitignore`
- [ ] 仓库可公开访问（Public）

## 3. GitHub Pages 配置

- [ ] `Settings -> Pages -> Source` 选择 `Deploy from a branch`
- [ ] 分支选择 `main`，目录选择 `/ (root)`
- [ ] 部署状态显示成功（绿色）
- [ ] 访问地址可打开：`https://<用户名>.github.io/<仓库名>/`

## 4. 上线后自测

- [ ] Chrome 桌面端可玩
- [ ] 手机浏览器可玩（至少 iOS/Android 各测 1 台）
- [ ] 首次点击后背景音乐可播放
- [ ] 刷新页面后高分仍保留（`localStorage`）
- [ ] 页面无 404 资源报错（控制台 Network 检查）

## 5. 发布文案与推广素材

- [ ] 准备一句话介绍（20~40 字）
- [ ] 准备 1 张封面图（1280x720 或 1080x1080）
- [ ] 准备 15~30 秒试玩视频
- [ ] README 在线链接已替换为真实 URL

## 6. 快速回滚预案

- [ ] 保留上一稳定版本 tag（如 `v1.0.0`）
- [ ] 若线上异常，回退到稳定提交并重新推送 `main`
- [ ] 回滚后再次执行“上线后自测”
